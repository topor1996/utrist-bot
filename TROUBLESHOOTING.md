# Решение проблем: Бот не отвечает

## 1. Проверьте логи на Railway

1. Откройте ваш проект на Railway
2. Перейдите на вкладку **"Logs"**
3. Посмотрите, есть ли ошибки

**Что искать:**
- ✅ Если видите "Бот запущен" или "Bot started" - бот работает
- ❌ Если видите ошибки - скопируйте их

**Типичные ошибки:**
- `Invalid token` - неправильный токен
- `Connection error` - проблемы с сетью
- `Module not found` - не установлены зависимости

## 2. Проверьте переменные окружения на Railway

1. Откройте ваш сервис на Railway
2. Перейдите на вкладку **"Variables"**
3. Убедитесь, что все переменные добавлены:

```
BOT_TOKEN=8506244542:AAHFZAl9YSz-eQ1qKDQQMtP6S7SnwbZJaj4
ADMIN_IDS=236217645
DATABASE_URL=sqlite:///bot_database.db
WORK_START_HOUR=9
WORK_END_HOUR=18
WORK_DAYS=1,2,3,4,5
```

**Важно:** 
- Токен должен быть БЕЗ пробелов
- ID должен быть БЕЗ пробелов

## 3. Проверьте статус деплоя

1. Перейдите на вкладку **"Deployments"**
2. Должна быть зеленая галочка ✅
3. Если красный крестик ❌ - нажмите "Redeploy"

## 4. Перезапустите сервис

1. Откройте Settings вашего сервиса
2. Нажмите **"Restart"** или **"Redeploy"**
3. Дождитесь успешного деплоя

## 5. Проверьте бота в Telegram

1. Убедитесь, что вы пишете правильному боту
2. Username: `@vashuristrus_bot`
3. Отправьте команду `/start` (не просто текст)

## 6. Проверьте команды бота в BotFather

1. Откройте @BotFather
2. Выберите вашего бота
3. Нажмите "Commands"
4. Убедитесь, что команды настроены (или настройте их):

```
start - Начать работу с ботом
admin - Админ-панель
```

## 7. Проверьте, что бот не заблокирован

1. В Telegram найдите вашего бота
2. Убедитесь, что бот не заблокирован
3. Если заблокирован - разблокируйте

## 8. Тестовая проверка

Попробуйте отправить боту:
- `/start` - должна быть реакция
- Любой текст - должна быть реакция или сообщение об ошибке

## Если ничего не помогает:

1. **Проверьте токен:**
   - В BotFather нажмите "API Token"
   - Убедитесь, что токен совпадает с тем, что на Railway

2. **Создайте новый токен:**
   - В BotFather: `/revoke` - отозвать старый токен
   - `/newtoken` - создать новый
   - Обновите `BOT_TOKEN` на Railway

3. **Проверьте логи детально:**
   - В Railway → Logs
   - Скопируйте последние 50 строк логов
   - Ищите ошибки Python

## Быстрая диагностика:

Выполните эти шаги по порядку:
1. ✅ Проверьте логи Railway
2. ✅ Проверьте переменные окружения
3. ✅ Перезапустите сервис
4. ✅ Проверьте бота в Telegram
5. ✅ Отправьте `/start`

Если после всех шагов бот не отвечает - пришлите скриншот логов из Railway.
