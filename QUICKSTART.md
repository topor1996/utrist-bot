# Быстрый старт

## 1. Получение токена бота

1. Откройте Telegram и найдите **@BotFather**
2. Отправьте команду `/newbot`
3. Придумайте имя для бота (например: "Ваш юрист СПб")
4. Придумайте username (должен заканчиваться на `bot`, например: `vash_urist_spb_bot`)
5. Скопируйте полученный токен (выглядит как: `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`)

## 2. Получение ID администратора

1. Найдите в Telegram бота **@userinfobot**
2. Отправьте команду `/start`
3. Скопируйте ваш **ID** (число, например: `123456789`)

## 3. Настройка проекта

1. Откройте файл `.env.example` и скопируйте его как `.env`:
   ```bash
   cp .env.example .env
   ```

2. Откройте `.env` и заполните:
   ```
   BOT_TOKEN=ваш_токен_от_BotFather
   ADMIN_IDS=ваш_telegram_id
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

## 4. Запуск

### Локально (для тестирования):
```bash
python bot.py
```

### На сервере (Railway/Render):

1. **Railway.app:**
   - Зарегистрируйтесь на https://railway.app
   - Создайте новый проект
   - Подключите GitHub или загрузите файлы
   - Добавьте переменные окружения из `.env` в настройках проекта
   - Railway автоматически запустит бота

2. **Render.com:**
   - Зарегистрируйтесь на https://render.com
   - Создайте новый Web Service
   - Подключите репозиторий
   - Укажите команду: `python bot.py`
   - Добавьте переменные окружения

## 5. Проверка работы

1. Найдите вашего бота в Telegram (по username, который вы указали)
2. Отправьте команду `/start`
3. Должно появиться приветственное сообщение с меню

## 6. Добавление дополнительных администраторов

В файле `.env` в `ADMIN_IDS` укажите ID через запятую:
```
ADMIN_IDS=123456789,987654321,555666777
```

## Проблемы?

- **Бот не отвечает:** Проверьте токен в `.env`
- **Ошибка при запуске:** Убедитесь, что установлены все зависимости (`pip install -r requirements.txt`)
- **Нет доступа к админ-панели:** Проверьте, что ваш ID указан в `ADMIN_IDS`
